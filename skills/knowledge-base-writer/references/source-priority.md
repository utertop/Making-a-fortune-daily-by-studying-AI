# Source Priority

Use this priority order when multiple sources exist:

1. Official repository: README, docs, examples, releases, changelog
2. Official product or company docs
3. Official blog post or launch post
4. Original paper or preprint
5. Author thread or announcement
6. Trusted secondary commentary

## Repo Checklist

For GitHub repositories, try to extract:

- What the project does
- Intended user
- Installation or quick start
- Main concepts or modules
- License
- Signs of activity: releases, commits, issues, PRs
- Risks: thin docs, unclear setup, broken examples, vague claims

## Conflict Rule

If sources conflict:

- Prefer the most direct primary source
- Prefer the newest official source when the change is temporal
- Mention the uncertainty instead of choosing silently

## Weak Source Set Rule

If only weak sources exist:

- Produce a partial note
- Say what is missing
- Keep TODO markers where evidence is not available
