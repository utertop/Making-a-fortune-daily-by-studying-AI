# Section Checklist

Use this checklist against `knowledge-base/templates/project-note.md`.

## Metadata

- Source
- Source type
- Signal score when available
- Confidence
- Tags

## TL;DR

Answer in 3-5 sentences:

- What is it
- Why is it relevant now
- Who should care

## Why It Matters

Answer:

- What trend or shift does it represent
- Why it matters for AI learning or engineering practice
- Whether it looks worth hands-on follow-up

## Quick Start

- Include only verified commands from source material
- If not verified, keep a TODO or mark as unverified

## Core Concepts

- 3-7 bullets
- Focus on mental model, not marketing language

## Architecture

Add Mermaid only when:

- there are multiple components or stages
- the flow is easier to understand visually

Use plain bullets instead when the system is simple.

## Evaluation Notes

Cover:

- Use case
- Docs quality
- Code quality
- Activity
- License
- Risk

## Hands-on Notes

- Record real setup results when tested
- Otherwise leave short TODOs for future execution

## Links

- Include the main repo or article
- Include docs and release notes when used
